# Type Coercion and Conversion

## What is Type Conversion?

> Explicit conversion done by the developer

```js
Number("10");   // 10
String(123);    // "123"
Boolean(1);     // true
```

## What is Type Coercion?

> Implicit conversion done by JavaScript

```js
"5" + 1; // "51"
"5" - 1; // 4
```

### String Coercion

If one operand is a string, `+` becomes concatenation.

```js
"5" + 1;     // "51"
"hello" + 5; // "hello5"
```
**Important contrast**

```js
"5" - 1; // 4
"5" * 2; // 10
```

> `-`, `*`, `/` force numeric conversion

> `+` is overloaded (string + number → string)


### Boolean Coercion

**All Falsy values, exhaustive list**:

```js
false
0
-0
0n
""
null
undefined
NaN
```

```js
Boolean("");     // false
Boolean("hi");   // true
Boolean(0);      // false
Boolean(100);    // true
```

### `==` vs `===`

#### Strict equality (===)
- No type coercion
- Both value and type must be the same

```js
1 === 1;       // true
1 === "1";     // false
0 === false;   // false
```

#### Loose equality (==)
- Performs type coercion if types differ

```js
1 == "1";      // true
0 == false;    // true
```

### The dangerous cases

#### Case 1: null and undefined

```js
null == undefined; // true
```

whereas

```js
null === undefined; // false
```

#### Case 2: Objects and primitives


```js
[] == ""; // true
[] == 0;  // true
[] == false; // true, first boolean coercion to 0, then [] is coerced to 0
```

- `[]` is coerced to `""` when compared with a string, and to `0` when compared with a number.
- false is coerced to `0` when compared with a number, and to `""` when compared with a string.
- This is a source of many bugs and confusion in JavaScript. Always prefer `===` to avoid these pitfalls!

#### Case 3

```js
"" == 0 // true, because "" is coerced to 0
```

#### Case 4

```js
"0" == false // true, because "0" is coerced to 0, and false is coerced to 0
```

## Common interview traps

```js
[] + {} // "[object Object]", because [] is coerced to "", and {} is coerced to "[object Object]"
{} + [] // 0, because {} is treated as an empty block, and +[] is coerced to 0
true + true; // 2
({} + []) // [Object Object]
"5" + 3 - 2; // "53" - 2 → 51, because "5" + 3 is "53", and then "53" is coerced to 53 when used with `-`
```

## More Examples

```js
"10" + 2; // "102", because "10" is a string, so + becomes concatenation
"10" - 2; // 8, because "10" is coerced to 10, and then 10 - 2 is 8
"10" * 2; // 20, because "10" is coerced to 10, and then 10 * 2 is 20
"10" / 2; // 5, because "10" is coerced to 10, and then 10 / 2 is 5
false == 0; // true, because false is coerced to 0
null == undefined; // true, because null and undefined are considered equal in loose equality
Boolean(" "); // ? true, because any non-empty string is truthy
```

```js
null + 1 // 1, because null is coerced to 0 when used with `+`
undefined + 1 // NaN, because undefined is coerced to NaN, Number(undefined) is NaN
"5" * "hello" // NaN, because "hello" cannot be coerced to a number. Number("hello") is NaN
null == 0 // false, because null is only loosely equal to undefined, and not to any other value
undefined == 0 // false, because undefined is only loosely equal to null, and not to any other value
null == undefined // true, because null and undefined are considered equal in loose equality
```

```js
//1. false to number, 0
//2. [] == 0
//3. [] -> primitive -> "" (empty string)
//4. "" == 0
//5. "" -> number -> 0
//6. 0 == 0 true
[] == false // true



[] == "" //true

[1] == 1 // true, because [1] is coerced to "1" (due to toString()), and then "1" is coerced to 1 when compared with a number
[1,2] == "1,2" // true, because [1,2] is coerced to "1,2"
[] + [] // "", because [] is coerced to ""
{} + [] // 0, because {} is treated as an empty block, and +[] is coerced to 0
[] + {} // "[object Object]", because [] is coerced to "", and {}
```

```js
[] == ![] // true, because ![] is false, and [] is coerced to "" which is falsy, so both sides are falsy and considered equal in loose equality
+true + +"5" // 6, because +true is 1, and +"5" is 5, so 1 + 5 is 6
"2" > "12" // true, because string comparison is lexicographical, and "2" comes after "1"
"2" > 12 //false, because "2" is coerced to 2, and then 2 > 12 is false
null > 0 // false, because null is coerced to 0 Number(null), and then 0 > 0 is false
null >= 0 // true, because null is coerced to 0 Number(null), and then 0 >= 0 is true
```

```js
[] + 1 + 2 // "12", because [] is coerced to "", so "" + 1 is "1", and then "1" + 2 is "12"
1 + 2 + [] // "3", because 1 + 2 is 3, and then 3 + [] is "3" (number + object with toString() results in string concatenation)
[1] + [2] // "12", because [1] is coerced to "1", and [2] is coerced to "2", so "1" + "2" is "12"
[1,2] + [3,4] // "1,23,4"
!!"false" == !!"true" // true, because both "false" and "true" are non-empty strings, so they are both truthy, and !!"false" is true, and !!"true" is true, so true == true is true
```
